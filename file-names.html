1. Basic Construction


Digital files that are part of our collection (either those created through digitization of our physical collection or born-digital items added to our collection) should be named according to the following pattern:
Collection ID_Series_Item#

Collection ID: Consists of three parts. See the Accession & Collection IDs pagefor further instructions.

Series:Record the name of theseries (or keyword for) within the larger collection to which it belongs if applicable.

Item #: Each item in a collection should have its own four digit item number.

For some kinds of materials, such as honors theses, other information may be included, such as author last name and year of creation.

2. Fronts and Backs of Items


I. If you are digitizing a photograph with writing onits backside, make a second scan of the back as well. Both scans will have the same item number. For the scan of the front side, add _f onto end ofthe file name. For the scan of theback side, add _b onto the end of the file name.

II. If you are digitizing a multipage document from a collection, digitize each page separately. Then structure the base file name as stated above. Each scan will have the same item number. Then add _#f or _#b as needed to the end of the file names. The # should reflect the page number of the original document, and the f and b should represent front or back of that particular page.

III. If you are digitizing a multipage document that is folded oddly, such as in Diagram A below, f and b might not be useful ways of reflecting how the scans relate to each other. In such cases, instead append _# to the end of the file names to reflect which page the scan represents within the original item.

3. General Best Practices for All File Names


One easy way to help ensure the longevity of all the files you create in your daily workis to avoid using spaces and symbols in your file names. Try using an underscore (_) instead of a space or capitalizing new words to enhance readability instead of using spaces.
